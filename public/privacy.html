WasmHatch Privacy Policy

Effective date: July 12, 2026

In short:

  • WasmHatch runs entirely in your browser. There is no WasmHatch server, no account, and no analytics.
  • Your files, workspace, and conversation history stay on your device unless you take an action that sends them somewhere.
  • Data leaves your browser only on your action: to the AI model provider you configure, directly to Google APIs under scopes you grant, or directly to Slack with a token you supply.
  • Access tokens and API keys are held in the memory of the open tab only — never stored, never sent to us.
  • Every write the agent makes is shown to you with its exact change, and workspace changes can be reverted; an optional Careful mode requires your approval before each write.

What WasmHatch is

WasmHatch (https://wasmhatch.com) is an open-source, client-side AI agent. You describe work in natural language and the agent does it with an AI model you choose, showing you the exact result of every change — a file diff, cell changes, or a document link — as it works, with one-click revert for workspace changes and an optional Careful mode that asks before each write. The application is a static web page: all application logic runs in your browser, and WasmHatch operates no backend server of its own.

Data that stays in your browser

The following data is stored only in your browser's own storage (Origin Private File System and localStorage) or in the memory of the open tab, on your device:

  • Files you import or create (CSV, XLSX, Markdown, JSON, text, scripts) and workspace snapshots.
  • Your conversation history and run journal.
  • API keys, OAuth access tokens, and other credentials you provide — these are held in tab memory only, are never written to disk, and disappear when the tab is closed.

WasmHatch never uploads this data to a WasmHatch server, because none exists. Content is transmitted elsewhere only in the specific, user-initiated cases described in the next section.

Data that leaves your browser — only when you act

Each of the following transmissions happens directly from your browser to the named third party, is initiated by your action, and is governed by that provider's own terms and privacy policy:

  • AI model providers. When you run an AI request using your own API key (for example OpenAI or Anthropic), the task text and the specific content you attach or approve for reading are sent to that provider. The app records this egress in a visible ledger. If you use Chrome's built-in on-device model instead, your content stays on the device.
  • Google APIs. When you connect a Google service, your browser obtains a short-lived access token through Google Identity Services. The token is held in tab memory only and attaches solely to requests your browser makes directly to googleapis.com under the scopes you granted. See Google user data below.
  • Slack. If you connect Slack with an Incoming Webhook URL or token you supply, requests go directly from your browser to Slack, and every message the agent posts is shown to you in the conversation as it happens. The webhook route is one-way: nothing is read from Slack.

A credential-broker layer inside the app keeps token and key text out of AI-model requests, generated scripts, and logs. Every external write — a spreadsheet edit, a document, a calendar event, a message — is surfaced to you with its exact content in the conversation, and an optional Careful mode requires your approval before each write.

Like any website, the static application files are delivered by a hosting provider, which may keep standard access logs (such as IP address and user agent) to operate the service. WasmHatch adds no analytics, tracking pixels, or telemetry of its own, and none of your workspace content is sent to the host.

What WasmHatch never does

  • No WasmHatch server storage: we cannot store your data because no WasmHatch server exists.
  • No analytics, tracking, or advertising of any kind.
  • No persistence of OAuth tokens or API keys — they live only in the memory of the open tab.
  • No sale of data, and no sharing of your data with anyone beyond the direct, user-initiated calls listed above.
  • No hidden writes: every change the agent makes to any system is shown to you with its exact content, workspace changes are revertible, and the optional Careful mode adds prior approval.

Google user data and Limited Use

When you grant Google scopes, WasmHatch accesses only the specific resources you point it at — a spreadsheet, document, or presentation you select or reference, or your calendar events — and uses that data solely to provide the feature you asked for in that session: displaying it, transforming it with your chosen AI model when you attach it, and preparing changes you review. WasmHatch stores Google user data only in your browser's local storage on your device, and shares it with no one; the only transmission of Google user data to a third party is when you explicitly attach content to a request to your own AI model provider, and that egress is recorded in the visible ledger. WasmHatch does not use Google user data for advertising and does not use it to train AI models.

WasmHatch's use and transfer to any other app of information received from Google APIs will adhere to the Google API Services User Data Policy, including the Limited Use requirements.

Deleting your data and revoking access

  • Local data: use the app's storage management to clear the workspace, or clear this site's data in your browser settings. Because everything is stored locally, this removes it completely — there is no server copy to request deletion of.
  • Google access: use Revoke Google access in the app, or remove WasmHatch at myaccount.google.com/permissions. Access tokens are never persisted, so closing the tab also ends the session.
  • Other credentials: AI provider keys, Slack webhook URLs, and Slack tokens are memory-only; close the tab to discard them, and revoke them at the issuing provider if desired.
