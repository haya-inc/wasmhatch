# v0.47.0 — A swarm you can share, worldwide

The hatchling swarm from the parallel track ships this week, complete
enough to be worth using: a small team of named agents, a shared ticket
board, tab-open auto work, MCP tools, and a pixel office to watch them in.
Any hatchling can now be packaged and shared — by file, by link, or
through an optional registry — and the whole UI speaks 18 languages.

## Highlights

- **Run a small team of hatchlings.** Up to eight agents, each with its
  own name, conversation, and isolated workspace, visualized as pixel
  characters at their desks. A shared ticket board (todo / doing / done /
  blocked) is the coordination surface, and any hatchling can be put on
  **auto work** — while the tab is open, it polls the board on the
  interval you set, with a visible cumulative run budget, automatic
  backoff after repeated failures, and one-click pause.
- **Share a hatchling as a portable agent.** Package one into a
  `wasmhatch.agent` file — its playbook and declared capabilities, never
  credentials — and export it, load one from a file or a trusted URL, or
  publish and discover them through an optional registry. Every import
  and publish stops at a plain-language preview card first; nothing
  hatches without a click, and unmapped capabilities simply grant
  nothing (fail-closed).
- **18 languages, fully translated.** Every UI string across the chat
  surface, tool summaries, and the pixel office now ships in 18 locales
  (see [docs/i18n.md](../i18n.md)), switchable in place with no reload.
- **Slack, both ways in.** Paste an Incoming Webhook for a 60-second
  one-channel setup, or connect a small internal Slack app's bot token
  to list public channels and post to any of them
  ([guide](../slack.md)). Post-only by design — nothing is read back.
- **Search the web through your own provider.** With a Claude or
  OpenRouter key, the model's own server-side search tool runs — no new
  key, no new CSP origin — and cited sources appear under the answer.
- **Run generated code, and keep your conversation.** `run_script` gives
  the chat agent the existing sandboxed QuickJS-in-Wasm worker for
  data-transform tasks (no network, DOM, or credential access), and the
  chat thread now survives a reload instead of resetting.
- **Hand a file to the agent, not the other way around.** The Google
  Picker opens as an in-conversation consent step — choose files and
  they become editable through the existing Google tools — and Google
  sign-in now renews silently a minute before it expires instead of
  interrupting a running turn.

## Under the hood

- Direct OpenAI chat moved to the Responses API, since reasoning models
  reject function tools on Chat Completions while reasoning is active;
  OpenRouter and Ollama stay on the OpenAI-compatible adapter.
- Four Google Sensitive scopes (Sheets, Docs, Slides, Calendar) are wired
  to real tools behind `VITE_GOOGLE_SENSITIVE_SCOPES` (default off), so
  Google's reviewers have something to watch the app use; production
  keeps launching on `drive.file` alone.
- The two legacy planner loops lost their anti-parity ceilings (6-call/
  8-turn/256 KB style hard stops) in favor of the same generous, visible
  soft budgets the chat loop already had.
- Twenty vendor remote MCP endpoints were measured from a real browser
  page and published as
  [docs/browser-mcp-servers.md](../browser-mcp-servers.md): five need no
  key at all, eight pass CORS with a bearer token, five refuse browsers
  outright.
- The chat sidebar gained a per-hatchling **run journal** — a passive,
  credential-redacted record of runs, tool calls, writes, and artifacts
  with one-click JSON export — the first flow ported off the retiring
  Operator page.
- Catalog integrity is now strictly checked after a translation script's
  interrupted write silently truncated the Polish catalog; every locale
  is diffed against the English message set on every check.

## For contributors and forks

- The capability vocabulary that fences a shared hatchling
  (`workspace.read`/`write`/`script`, `artifacts`, `tickets`, `google`,
  `google.sensitive`, `slack`, `mcp`) is the seam to extend if you add a
  new connector — see
  [docs/portable-agents.md](../portable-agents.md#runtime-model--hatchlings-are-the-unit-of-sharing).
- Good-first-issues are still open (#17, #27, #28, #29) and review
  turnaround stays a 48-hour goal.
- Fork note: a self-hosted registry only needs `VITE_REGISTRY_URL` set at
  build time; without it the sidebar's publish/load-by-URL controls stay
  hidden and `.agent` files still travel by hand.
